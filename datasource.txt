StayAnalytics — Data sources
=============================

https://www.kaggle.com/datasets/amaymishra11/ott-movies-and-series-dataset-ml-and-nlp-ready


1. Content catalog (2,500 rows)
-------------------------------
File: ott_movies_clean_unique.csv (kept in the project root for
      reproducibility).

Columns: content_id, title, type, genre, platform, country, language,
release_year, duration_minutes, rating, votes, weighted_rating,
engagement_score, popularity_score, trending_score, tags,
combined_features, description, poster_url.

Origin: synthetic OTT-platform dataset modeled after public Kaggle
"Streaming Service Movies & TV Shows" releases. Each row represents one
movie or series available on Netflix, Prime Video, or Hotstar. The
`poster_url` column contains a placeholder image URL (no real artwork
is shipped to avoid copyright issues).

2. Demo users (3 rows)
----------------------
Hardcoded in scripts/generate_movies_sql.py with BCrypt hashes
pre-computed at strength 12 so data.sql is reproducible without
network access:

    admin / Demo1!  (ADMIN)
    alice / Demo2!  (USER)
    bob   / Demo3!  (USER)

3. Reviews (5,000 rows) and watchlist (500 rows)
------------------------------------------------
Generated by scripts/generate_movies_sql.py. The script:
  - seeds Python's random with 4400 (deterministic output);
  - picks (user_id, content_id) pairs uniformly at random, ensuring
    no duplicates (matches the UNIQUE constraint on review);
  - draws a 1-10 rating from a positively-skewed distribution
    (most ratings between 7 and 10) so the analytics dashboard shows
    realistic averages instead of a uniform 5.5;
  - picks a watchlist status uniformly from
    {PLAN_TO_WATCH, WATCHING, WATCHED}.

How to regenerate data.sql
--------------------------
    python3 -m pip install bcrypt        # only needed once
    python3 scripts/generate_movies_sql.py

The script reads ott_movies_clean_unique.csv and rewrites data.sql
in place.
